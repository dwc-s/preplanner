# Development

## Setup

**Fastest** (macOS/Linux) — `./install.sh` does everything below (creates the venv,
installs deps, writes a `.env` with a generated `SECRET_KEY`, applies migrations) and
offers to create an admin or load demo data. Re-runnable; never clobbers `.env`/the DB.

Manual:

```bash
git clone https://github.com/dwc-s/preplanner && cd preplanner
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python run.py          # applies migrations on startup, serves 127.0.0.1:5000
```

`FLASK_APP=run` is provided by `.flaskenv` (loaded by the `flask` CLI via
python-dotenv), so `flask …` commands work without exporting anything.

For sample data: `flask seed-db` (or `python seed.py`, which also migrates). The demo
login is `admin@example.com` / `changeme`.

**Optional OCR:** image text search needs a `tesseract` binary on PATH
(`brew install tesseract` / `apt install tesseract-ocr`). Without it, uploads still
work and image OCR just stays queued.

## Running & the service worker

`python run.py` runs in debug (auto-reload). The service worker is **skipped on
`localhost`/`127.0.0.1`** (`base.html` unregisters it and clears its caches), so CSS/JS
edits appear on a plain refresh — no cache-version bump needed locally. In production,
**bump `APP_CACHE` in [`app/static/sw.js`](../app/static/sw.js)** whenever shell
JS/CSS changes, or returning clients keep the old cached assets.

## Tests

```bash
pytest
```

If bare `pytest` fails to import `app` (e.g. it resolves to a system/conda Python),
run it through the venv with the repo on the path:

```bash
PYTHONPATH=. python -m pytest -q
```

Each test uses a throwaway file-based SQLite DB and a temp `UPLOAD_FOLDER`, so nothing
touches `instance/`. CSRF and rate-limiting are off in the default test config; one
test re-enables CSRF to prove it's wired. OCR-dependent tests `skipif` no `tesseract`.

Coverage spans auth + cross-tenant isolation, pre-plan CRUD, dashboard/review,
sandbox, the builder + asset library (incl. GPS and OCR extraction), offline sync +
conflicts, GIS import, and rate limits.

## Migrations

Schema changes are Alembic migrations via Flask-Migrate:

```bash
# 1. Edit the model in app/models.py
flask db migrate -m "what changed"     # autogenerate
# 2. REVIEW + adjust the generated file (see gotchas), then:
flask db upgrade
```

**Always review the autogenerated migration.** Two gotchas bite here (both have
precedent in `migrations/versions/`):

1. **`NOT NULL` column on an existing table needs a `server_default`** to backfill
   rows, or the upgrade fails. Add one, e.g.
   `sa.Column('status', sa.String(20), nullable=False, server_default='draft')`.
2. **SQLite batch mode requires *named* foreign-key constraints.** Autogenerate emits
   `create_foreign_key(None, …)`, which raises "Constraint must have a name" — give it
   a name like `'fk_occupancy_created_by'` (and match it in `downgrade`).

New models get `CREATE TABLE` (inline FKs), which is fine as-is.

## Adding a syncable (offline) entity

To make a record editable offline, add an `EntitySpec` to `SYNCABLE` in
[`app/sync.py`](../app/sync.py) (fields, optional `parent`/`parent_fk`, optional
`on_create` hook), add it to `APPLY_ORDER` (parents before children), give the model a
`uuid` + `updated_at`, and register a Dexie table in
[`store.js`](../app/static/js/store.js). Server-managed fields (like `created_by`,
`status`) are set via `on_create` and kept **out** of the field list so client edits
never overwrite them.

## Conventions

- **Tenant isolation** goes through `dept_query` / `get_owned` in
  [`app/scoping.py`](../app/scoping.py) — never query a dept-scoped model directly in a
  route. There's an isolation test guarding this.
- **Controlled vocabularies** (occupancy types, ranks, statuses, element kinds…) live
  as lists/dicts in `models.py` and are exposed to templates by the `inject_choices()`
  context processor in the factory.
- **Autosave over Save buttons.** Record-editing forms opt into
  [`autosave.js`](../app/static/js/autosave.js) with a `data-autosave` attribute
  (debounced POST + `X-Autosave: 1` → route returns JSON `{ok}`); see
  [ARCHITECTURE.md](ARCHITECTURE.md#autosave). Keep explicit buttons only for
  publish / submit / delete actions.
- **No front-end build step.** Vendored libraries live in `app/static/vendor/` (and the
  PT Serif webfont in `app/static/fonts/`); add new ones there rather than a CDN
  (offline + AGPL friendliness). Bump `APP_CACHE` in `sw.js` when shell JS/CSS/fonts
  change so returning PWA clients refresh.
- **Pure-Python deps only** where feasible (the app targets free hosting with no
  system libraries beyond an optional `tesseract`).

## CLI commands

| Command | Does |
|---------|------|
| `flask db upgrade` / `flask db migrate -m "…"` | Apply / create migrations |
| `flask create-admin` | Create a department + first admin (interactive) |
| `flask seed-db` | Load the demo department + sample data (idempotent) |
| `flask ocr-pending [--limit N]` | OCR queued photos |
| `flask purge-sandboxes [--max-age-hours N]` | Delete expired demo sandboxes |
